# Support

Use the shortest route that matches the problem:

- Ask usage and methodology questions in GitHub Discussions.
- Report reproducible defects with the bug issue form.
- Propose product or protocol changes with the feature or research issue form.
- Report vulnerabilities privately through the Security tab.

Include `uv run cib doctor` output for setup problems, but review it before
posting. Never attach auth files, private manifests, or raw result directories.

CIB is maintained as an open-source project without a response-time SLA.
