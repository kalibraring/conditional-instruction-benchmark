# Governance

CIB uses a maintainer-led model during its pre-1.0 phase.

Mohamad Kamar is the initial maintainer and owns release decisions, security
coordination, protocol compatibility, and final claim boundaries. Contributors
influence direction through issues, design proposals, code review, experiments,
and reproducible evidence.

The maintainer may merge routine fixes after CI passes. Changes to causal arms,
ground truth, scoring, evidence schemas, missingness, recovery policy, or agent
comparability require a public design issue and an explicit decision record.
Security matters may remain private until coordinated disclosure.

The project will revisit governance when regular contributors emerge or when a
stable 1.0 protocol needs shared stewardship.
